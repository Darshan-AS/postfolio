# Session Log: 2026-05-10 (Session 4)

## Goal
Fix the design issue where KVP's "Term Years" and "Money Multiplier" showed different numbers due to free-text custom tenure inputs allowing users to override derived calculations.

## Changes Made
- **Domain Modeling**: Replaced the `isFixedTenure` boolean on scheme extensions with a strict `TenureInputType` enum (`singleFixed`, `fixedOptions`, `derived`) to properly represent Indian Post Office scheme rules.
- **UI Refactoring (`AppDurationInput`)**: 
  - Refactored `AppDurationInput` to use `TenureInputType`.
  - Removed `_buildCustomTenure` completely to eliminate arbitrary free-text tenure fields.
  - Added `_buildDerivedTenure` to show read-only dynamically calculated values for derived schemes (like KVP).
- **Core Math**: Exposed `calculateKvpTermMonths(interestRate)` in `ProjectionCalculator` to compute the derived maturity length in months cleanly.
- **Form Synchronization**:
  - `one_time_deposit_form_screen.dart`: Auto-synced `selectedTermYears` and `selectedTermMonths` in a `useEffect` based on the interest rate so the mathematically correct derived value is saved to the backend.
  - Supplied `derivedString` (e.g., "9 Years & 7 Months") from the projection's note directly to the duration UI.
- **Model Validation**: Updated validation rules in `one_time_deposit_model.dart` and `recurring_deposit_model.dart` to rely on `tenureInputType != TenureInputType.derived`.
- **Mocks**: Adjusted `fake_data_source.dart` to calculate correct term years/months when generating mock KVP data.

## Next Steps
- Implement early withdrawal/premature closure calculation logic for all applicable schemes (TD, MIS, SCSS, KVP, MSSC, PPF).
