# Session Log: 2026-05-09 (Session 3)
## Goal
Review the Demo Mode implementation for any broken conventions or standards, and remove the environment variable dependency as requested by the user.

## Changes Made
- Identified that the `USE_FAKE_DATA` environment variable configuration was requested to be removed.
- Completely removed the environment variable fallback (`const envDemoMode = bool.fromEnvironment...`) from `StorageService`, making `StorageService` strictly read from `SharedPreferences` as intended by the persistence standard.
- Removed the environment variable specific `launch.json` profile since the user no longer wanted it.
- Extracted the Demo Mode banner logic from `DashboardScreen` into a dedicated `DemoBanner` widget (`lib/core/widgets/demo_banner.dart`).
- Integrated `DemoBanner` at the top of the body inside `MainShellScaffold` for both mobile and tablet layouts, ensuring the banner persists and is visible across all authenticated screens.
- Fixed a layout issue where the child `AppBar` rendered excessive padding. Wrapped the `navigationShell` inside `MainShellScaffold` with `MediaQuery.removePadding(removeTop: true)` when demo mode is active, which tells the descendant AppBars not to reserve space for the status bar (since the banner handles it).
- Identified that the `FakeCustomerRepository`, `FakeOneTimeDepositRepository`, and `FakeRecurringDepositRepository` could leak `StreamController`s. Added a `dispose` method to them and hooked it into the provider's `ref.onDispose` to prevent memory leaks during toggles between real and demo data.
- Found hardcoded strings in `LoginScreen` ("Try Demo Mode") and `DashboardScreen` ("Demo Mode Active. Data is not saved to the cloud."). This broke the app's standard convention of using `slang` for all user-facing strings.
- Added `tryDemoMode` and `demoModeActive` to `lib/i18n/en.i18n.yaml` under their appropriate categories and ran `dart run slang` to regenerate `strings.g.dart`.
- Replaced the hardcoded strings with `t.auth.tryDemoMode` and `t.common.demoModeActive`.
- Updated `tasks.md` to remove the mention of "via environment variables".
