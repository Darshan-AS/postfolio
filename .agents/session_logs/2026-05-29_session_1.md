# Session Log: 2026-05-29 (Session 1)

## Goal
Fix a startup crash / debugger pause occurring when using the Firebase emulator, specifically halting at `element.handleCreate(ref, build);` within `customers_controller.g.dart`.

## Actions Taken
- Investigated the Riverpod provider initialization for `customerRepositoryProvider`, `oneTimeDepositRepository`, and `recurringDepositRepository`.
- Discovered that when `userId == null` (i.e., user is unauthenticated during the initial app load), the repositories were returning a class that emitted a `Failure('User is not authenticated')`.
- The corresponding controllers (e.g., `CustomersController`) were designed to intercept `Failure` from the repository stream and `throw Exception(error)`.
- This thrown exception caused the IDE's debugger to pause execution on "Uncaught Exception" during startup before `GoRouter` could finish redirecting to the login screen.
- Created `EmptyCustomerRepository`, `EmptyOneTimeDepositRepository`, and `EmptyRecurringDepositRepository` implementations.
- Modified these empty repositories to emit `Success([])` instead of `Failure` for their respective `watch` streams when unauthenticated.
- This approach respects the functional pattern by avoiding thrown exceptions during expected unauthenticated states, allowing `GoRouter` to handle the routing seamlessly to the login page.
- Validated via `flutter run -d chrome --dart-define=USE_EMULATOR=true` that the app successfully launches to the login screen without hitting an unhandled exception.
