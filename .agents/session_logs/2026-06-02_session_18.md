# 2026-06-02 Session 18: Enhanced Release Preparation

## Overview
Upgraded the "Release Manager" skill and agent to perform data-driven versioning and changelog generation by analyzing git commit history.

## Changes Made
### Agent Customization
- **`SKILL.md`**: Updated the "Prepare Release" procedure with explicit steps to:
    1. Identify the last git tag.
    2. List commits since the last tag.
    3. Analyze commits (Major/Minor/Patch) based on prefixes like `Feat:`, `Fix:`, etc.
    4. Automatically draft summarized CHANGELOG entries.
- **`release-manager.agent.md`**: Updated the workflow to prioritize commit analysis as the first step in the release cycle.

## Decisions & Conventions
- **Data-Driven Versioning**: Instead of guessing the release type, the agent must now use `git log` to provide an evidence-based recommendation for the next version number.
- **Semantic Commit Awareness**: The agent is now instructed to recognize Conventional Commits to distinguish between features, fixes, and breaking changes.
