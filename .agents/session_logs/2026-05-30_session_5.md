# Session Log: May 30, 2026

## Summary
Improved UI/UX consistency across the Search, Filter, and Sort mechanisms on entity listing screens, as per user request to standardize behavior.

## Changes Made
- Refactored `AppSortBottomSheet` from a `StatelessWidget` to a `HookWidget` to maintain local state, requiring users to explicitly tap 'OK' (Apply) before the selected sort option propagates to the Riverpod providers.
- Standardized headers across `AppSortBottomSheet` and `AppFilterBottomSheet` to include the title on the left and a standard Close icon (`HugeIcons.strokeRoundedCancel01`) on the right.
- Moved the 'Clear Filters' button in `AppFilterBottomSheet` to the sticky bottom row, adjacent to the 'OK' button.
- Added `AnimatedSize` wrappers around `AppSearchBar` in `OneTimeDepositsScreen`, `CustomersScreen`, and `RecurringDepositsScreen` to prevent jarring UI shifts when toggling the search bar.
- Introduced a `Badge` wrapper around the Sort icon in App Bars that displays a visual indicator (red dot) when a non-default sorting option is currently active.
- Updated `.agents/progress.md` with the completed UI consistency tasks.
