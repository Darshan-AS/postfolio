Research routing setup and define Navigation Policy.

## Actions Taken
- Researched Flutter routing packages (`go_router`, `auto_route`, `routemaster`).
- Concluded that `go_router` combined with `go_router_builder` and `StatefulShellRoute` is the best holistic routing solution, offering the best balance of web deep linking, predictive back gesture, and stateful nested navigation support.
- Defined a strict `go` vs `push` Navigation Policy to balance web URL predictability and native mobile UX.

  - `push()`: Use for vertical hierarchical movement, Master-Detail flows, and Forms (to support native back gestures).
