# Session Log: 2026-05-02
## Session 4: UI Cleanup - Date Formatting

### Changes Made
- **Centralized Date Formatting:** Created `DateTimeFormatting` extension in `lib/core/extensions/date_time_extension.dart` to provide consistent, unambiguous date strings across the application.
- **Slang Integration:** Added new translation keys in `en.i18n.yaml` under the `format` node (`date`, `dateCompact`, `dateTime`) and re-generated `strings.g.dart`.
- **Refactoring:** Replaced all hardcoded `DateFormat` and `MaterialLocalizations.of(context).formatCompactDate` usages with `.toAppFormat()` in:
  - `CustomerFormScreen` & `CustomerDetailScreen`
  - `OneTimeDepositFormScreen`, `OneTimeDepositDetailScreen` & `OneTimeDepositCard`
  - `RecurringDepositFormScreen`, `RecurringDepositDetailScreen` & `RecurringDepositCard`
  - `InvestmentProjectionCard`
- **Docs Update:** Marked "UI Cleanup - Date Formatting" as completed in `tasks.md`. Added a new potential feature task for "Relative Time Formatting".

### Decisions Made
- Chose to use a static Slang string definition (`dd MMM yyyy`) instead of device locale-dependent `MaterialLocalizations` to ensure visual consistency and zero ambiguity in dates for a financial app (e.g. avoiding the 10/12/2026 month/day confusion).
