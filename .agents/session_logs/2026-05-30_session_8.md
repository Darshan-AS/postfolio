# 2026-05-30 Session 8: Routing Fix

## Goal
Fix `GoRouter` exception `_AsyncCompleter.complete` (ImperativeRouteMatch) when pressing the system back button on Android.

## Changes
- Identified that the app was using `.push(context)` for routes that are defined within the `GoRouter` declarative tree.
- Migrated all primary navigation calls from `.push()` to `.go()` in:
    - `CustomersScreen`
    - `CustomerDetailScreen`
    - `OneTimeDepositsScreen`
    - `OneTimeDepositDetailScreen`
    - `RecurringDepositsScreen`
    - `RecurringDepositDetailScreen`
- This ensures that `GoRouter` manages the navigation stack declaratively, avoiding conflicts between the `Navigator`'s imperative stack and `GoRouter`'s internal state.
- Updated `AGENTS.md` and `.agents/conventions.md` to reflect the use of `go_router_builder` and the preference for `.go()` navigation.
- Fixed analysis errors in `MainShellScaffold` by importing necessary route classes and passing `BuildContext` to the `_goBranch` helper method.

## Results
- The "ImperativeRouteMatch.complete" exception is resolved as `go_router` no longer tracks these transitions as imperative overlays.
- System back button now correctly triggers declarative location updates in `go_router`.
