# Session Log: 2026-06-01

## Context
Refactoring app theming to adhere strictly to Material 3 design conventions.

## Changes Made
- **`AppColors` Refactor**: Stripped out legacy static semantic colors (`surface`, `error`, `textPrimary`, etc.), keeping only the core brand colors (`primary`, `primaryDark`, `accent`) as seeds for the `ColorScheme`.
- **`AppWidgetThemes` Modernization**: Updated all widget theme configurations (`AppBarTheme`, `CardTheme`, `FloatingActionButtonThemeData`, etc.) to dynamically accept and consume a `ColorScheme`. Removed hardcoded font sizes (e.g. `AppDimensions.fontLg`) to allow Material 3 typography scaling (`TextTheme`) to behave natively.
- **`AppTheme` Update**: Wired `lightTheme` to pass the generated `colorScheme` into all widget themes, and fully implemented a `darkTheme` configuration.
- **`main.dart`**: Plumbed the new `darkTheme` property into the top-level `MaterialApp.router`.
- **UI Adjustments**:
  - Replaced `AppColors.textSecondary` with `colorScheme.onSurfaceVariant` in `AppInputDecoration`.
  - Replaced hardcoded `AppDimensions.fontXs` scaling in `detail_components.dart` with `labelSmall` scaling.
  - Eliminated static `AppColors.*` references in `run_migration.dart` in favor of standard `Colors` constants and `Theme.of(context).colorScheme`.
