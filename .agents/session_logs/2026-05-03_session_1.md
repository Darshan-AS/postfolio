# Session Log: 2026-05-03

## Focus
Implemented Firebase Authentication with Google Sign-In, adhering strictly to the architecture conventions (Riverpod, Freezed, Functional principles).

## Changes Made
1. **Dependencies:** Added `firebase_auth` and `google_sign_in` (7.0.0+) to `pubspec.yaml`.
2. **Platform Setup:** Configured iOS `Info.plist` with `CFBundleURLTypes` for the Google Client ID.
3. **Domain Layer:** 
   - Created `AppUser` using `@freezed sealed class`.
   - Created `AuthState` using `@freezed sealed class` (with explicitly named subclasses `AuthStateLoading`, `AuthStateAuthenticated`, etc. for Dart 3 switch matching).
4. **Data Layer:** Built `AuthRepository` implementing `_googleSignIn.authenticate()` and `GoogleAuthProvider.credential` following the new `google_sign_in` 7.x singleton API format.
5. **Presentation Layer:** Built `AuthController` via `@Riverpod(keepAlive: true)` and `LoginScreen` utilizing pure state transitions (`loading`, `error`, `authenticated`).
6. **Routing:** Configured `GoRouter` in `app_router.dart` with a `ValueNotifier` listener for reactive redirects on authentication status changes. Added `LoginRoute`.
7. **Initialization:** Added `Firebase.initializeApp` to `main.dart`.
8. **Documentation:** Updated `.agents/tasks.md` and `.agents/progress.md` with the new Firebase Authentication progress.

## Decisions
- Maintained the Freezed pattern using public factory names instead of private unions to enable proper Dart 3 switch expression pattern matching outside the file scope.
- Adapted to the new `google_sign_in` 7.x API singleton (`GoogleSignIn.instance.authenticate()`) rather than downgrading the library.

## Bug Fixes
- **Web Google Sign-In:** Fixed an issue where Google Sign-In failed on Web. 
  - Passed the dedicated Web Client ID (`143378829514-84as322oeajpt990vu2fil56lvs4l9b7.apps.googleusercontent.com` from `google-services.json` `client_type: 3`) to `_googleSignIn.initialize()` conditionally when `kIsWeb` is true.
  - Added the `<meta name="google-signin-client_id" ...>` tag in `web/index.html` as a fallback.
  - Adapted the Web `googleAuth.accessToken` extraction to securely ignore token fetching on web (where `authenticate` returns only an `idToken` under GIS) and rely on the `idToken` which is sufficient for Firebase Auth.

- **Web Google Sign-In Reprise:** Re-addressed the web issue where `_googleSignIn.authenticate()` threw an `UnimplementedError` because Google Identity Services strictly requires its own native web UI (`renderButton`).
  - Switched exclusively to Firebase Auth's native Web popup handling via `FirebaseAuth.instance.signInWithPopup(GoogleAuthProvider())` when running on `kIsWeb`. 
  - Kept the native `google_sign_in` plugin flows explicitly isolated for Mobile targets. 
  - This natively links Firebase without the need for separate JS widgets!

- **Cleanup:** Removed the previously created `ConditionalSignInButton` widgets that depended on `google_sign_in_web/web_only.dart`. We now strictly use a native Flutter `FilledButton` across all platforms since Firebase Auth handles the native popup cleanly on web.

- **Android Google Sign-In Fix:** Fixed "Account reauth failed" on Android. 
  - `google_sign_in` 7.0.0+ uses Android's new Credential Manager. To receive an `idToken` (which Firebase requires), we MUST explicitly pass the Web Client ID as `serverClientId` to `_googleSignIn.initialize()`.
