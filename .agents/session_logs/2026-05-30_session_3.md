# Session Log: 2026-05-30 - Session 3

## Changes Made
- Migrated the custom search bar to the official Material 3 `SearchBar` widget.
- Implemented `SearchBarThemeData` in `AppWidgetThemes` to ensure the official widget matches the app's flat design (elevation 0, radius 12.0, specific colors).
- Updated `AppSearchBar` to include a "Cancel" button and an `onClose` callback, using standard Material 3 icon sizes (24.0).
- Integrated the new `AppSearchBar` with `onClose` logic across `CustomersScreen`, `OneTimeDepositsScreen`, and `RecurringDepositsScreen`.
- Fixed a critical "Type mismatch between hooks" error in `AppSearchBar` caused by incorrectly nesting `useRef` inside `useEffect`. Moved all hooks to the top level and added `useListenable` to ensure the UI updates when typing.
- Fixed the closing logic: clicking "Cancel" now correctly hides the search bar and resets the search criteria in Riverpod.

## Decisions Taken
- Used the official `SearchBar` but stripped its "pill" shape and elevation to maintain the project's consistent "ledger" aesthetic.
- Opted for a `TextButton` for the "Cancel" action outside the search bar to follow mobile system patterns and provide a clear tap target.
- Maintained the custom debouncing logic (300ms) within the search controller listener using `Timer`.
