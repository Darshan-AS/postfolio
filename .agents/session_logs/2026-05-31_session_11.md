# Session Log: 2026-05-31

## Goal
Implement a quick UI action to mark recurring and one-time deposits as closed or active without navigating to the full edit screen.

## Changes Made
1. **State Management**:
   - Added `toggleDepositStatus(String id, DepositStatus newStatus)` in both `RecurringDepositsController` and `OneTimeDepositsController`.
2. **Core UI Widgets**:
   - Added `customActions` to `EntityDetailScaffold` to inject action buttons into the AppBar.
   - Refactored `AppDialogs.showError` and added `AppDialogs.confirmAction` for versatile dialog confirmations.
3. **List View Cards**:
   - Added `onToggleStatus` callback to `RecurringDepositCard` and `OneTimeDepositCard`.
   - Cards now display a "Close" or "Reopen" action in their PopupMenu based on the deposit's current status.
4. **Detail Screens**:
   - Injected a toggle status `IconButton` in the AppBar of both `RecurringDepositDetailScreen` and `OneTimeDepositDetailScreen`.
5. **Localization (Slang)**:
   - Added required keys to `lib/i18n/en.i18n.yaml` in the `common`, `recurringDeposits`, and `oneTimeDeposits` sections.
   - Updated texts to be shorter ("Close" and "Reopen").

## Conventions Followed
- Riverpod state updates handled cleanly via the repositories.
- `AppDialogs` abstraction maintained for dialogs.
- `HugeIcons` used consistently with `AppDimensions.iconMd`.
- Slang used exclusively for hardcoded texts.
