# Session Log: 2026-05-09 (Session 2)
## Goal
Investigate and fix the app crashing when logging out after a successful login (either real or demo).

## Changes Made
- Identified that `GoRouter` was throwing a `Hero` animation exception (`Hero._allHeroesFor`) during the routing transition on logout. This happens because `FloatingActionButton.extended` assigns a default `heroTag` automatically. When the navigation stack is violently replaced upon logging out, multiple routes attempting to match the default Hero tags were crashing the Flutter framework during `visitChildren`.
- Explicitly assigned `heroTag: null` to the `FloatingActionButton.extended` widgets inside:
  - `CustomersScreen`
  - `RecurringDepositsScreen`
  - `OneTimeScreen`
- This ensures that when GoRouter wipes the route history and returns the user to the `LoginScreen`, no Hero transitions are mistakenly attempted, fixing the crash entirely.
